# 有序日常 — 命令行启动使用说明

## 项目简介

有序日常是一款 uni-app（Vue 3）跨平台应用。项目由前端 H5 页面和后端 API 服务组成，后端通过 Express 同时托管前端静态页面和 REST API。

---

## 一、环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Node.js | 18.x+ | 后端运行环境 |
| MySQL | 5.7 / 8.0 | 数据库（可选，游客模式不需要） |

---

## 二、快速启动（仅需 3 步）

### 第 1 步：创建 dist 目录（链接前端构建产物）

项目已内置构建好的 H5 前端页面，位于 `unpackage/dist/build/web/`。需要创建一个 `dist/` 目录让后端能找到它：

```bash
# 在项目根目录执行（创建符号链接）
# Windows (以管理员身份运行终端):
mklink /D dist unpackage\dist\build\web

# Mac / Linux:
ln -s unpackage/dist/build/web dist
```

> 如果无法创建符号链接，也可以直接复制：
> ```bash
> # Windows
> xcopy /E /I unpackage\dist\build\web dist
>
> # Mac / Linux
> cp -r unpackage/dist/build/web dist
> ```

### 第 2 步：安装后端依赖并配置环境变量

```bash
cd server
npm install
```

编辑 `server/.env`，在文件末尾添加一行，指定前端静态文件路径：

```env
STATIC_DIR=../../dist
```

### 第 3 步：启动服务

```bash
cd server
npm run dev
```

看到以下输出表示启动成功：

```
Serving static files from: D:\...\有序日常\dist
HTTP server listening on 5080
```

在浏览器访问 **http://localhost:5080** 即可看到应用。

## 四、数据库初始化（可选）

游客模式下数据存储在浏览器本地，无需数据库。如需使用登录、云同步功能，则需初始化数据库：

```bash
# 确保 MySQL 已启动，然后：
cd server
node scripts/init_db.js
# 成功输出 → Schema applied successfully.
```

数据库配置在 `server/.env` 中：

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=yxrc
DB_PASSWORD=19900108
DB_NAME=youxurichang
```

## 五、命令速查

```bash
# ===== 安装依赖 =====
cd server
npm install

# ===== 启动服务（开发/生产相同） =====
npm run dev          # 开发模式
npm start            # 生产模式（两者执行的是同一个命令）

# ===== 数据库 =====
node scripts/init_db.js    # 初始化/重置数据库表

# ===== 前端重新构建（需 HBuilderX） =====
# 菜单栏 → 发行 → 网站-H5手机版
# 构建产物目录：unpackage/dist/build/web/
# 重新构建后重新执行第 1 步更新 dist 目录即可
```

## 七、常见问题

**Q: 启动后页面空白？**
检查终端是否有 `Serving static files from: ...` 这行输出。如果没有，说明 `dist/` 目录不存在或路径不对——确认第 1 步已执行。

**Q: 端口被占用？**
修改 `server/.env` 中的 `PORT=5080` 为其他端口（如 `3000`）。

**Q: 如何让前端修改实时生效？**
前端源码修改后需在 HBuilderX 中重新执行「发行 → 网站-H5手机版」，然后更新 `dist/` 目录。纯命令行环境无法做到前端热更新。

**Q: 没有 MySQL 能用吗？**
可以。游客模式下所有数据存在浏览器 localStorage 中，API 调用会失败但不影响基本使用。
