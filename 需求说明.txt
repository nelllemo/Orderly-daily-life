1.3 软件项目概述
•项目名称：有序日常
•项目简称：YXRC
•目标用户：学生群体（大学生为主）和20-30岁年轻上班族
•开发单位：个人开发团队
•产品定位：集智能日程管理、定时提醒、私密生活记录、AI智能辅助于一体的轻量化综合生活管理工具
•核心价值：解决用户待办分散易遗漏、手动录入繁琐、缺乏私密记录空间、现有工具臃肿等痛点，帮助用户高效管理时间，有序规划生活

1.5 定义
•日程：用户需要在特定时间完成的任务或活动，包含标题、时间、地点、备注、优先级等信息
•待办：尚未完成的日程
•动态：用户发布的个人生活记录，仅自己可见
•AI智能日程生成：通过自然语言处理技术，将用户的文字或语音描述转换为标准日程
•MVP：最小可行产品，包含核心功能的最早可发布版本
•晨间推送：每日固定时间自动推送当日待办清单的通知功能
•日程挂载：将个人动态与对应日程进行关联，实现任务与生活记录的双向绑定

2. 软件的一般性描述
2.1 软件产品与其环境之间的关系
"有序日常"APP支持两种使用方式：
1.Android原生APP：运行在Android 8.0及以上系统的智能手机上
2.H5网页版：通过手机浏览器访问服务器IP:端口使用全部功能，无需安装APP
与外部环境的交互如下：
•调用手机系统功能：通知、语音识别、相机、相册、日历、震动、铃声（原生APP）
•H5网页版：通过浏览器访问，支持相机拍照上传、本地图片选择，部分系统级功能受限
•网络交互：通过HTTP/HTTPS协议连接后端服务器，进行数据同步和AI请求，API支持同源部署和跨域访问
•第三方服务集成：调用豆包大模型API实现智能日程生成，调用微信登录服务实现用户认证
•数据隔离：不与其他第三方APP进行数据互通，所有用户数据独立存储在本系统中
2.2 限制与约束
•系统版本限制：Android原生APP仅支持Android 8.0及以上版本；H5网页版支持主流移动浏览器（Chrome、Safari、Edge、微信内置浏览器等）
•网络限制：AI功能和云端数据同步需要联网，离线状态下可查看和编辑本地数据（使用浏览器本地存储），联网后自动同步到云端数据库
•存储限制：为每个用户提供1GB免费云端存储空间，用于存储动态图片和视频（MVP版本仅支持图片）
•功能限制：MVP版本不支持团队协作、共享日程、数据导入导出、多设备同步（后续版本迭代）；H5网页版部分原生功能（系统通知推送、震动、铃声）受限
•时间限制：项目总周期为2个月，必须在2026年7月10日前完成MVP上线
•技术栈限制：前端采用hubuilderx框架（Vue3），后端采用Node.js+Express，数据库采用MySQL；H5网页版通过uni-app H5模式构建，与Express后端同源部署
2.3 假设与前提条件
•用户拥有符合系统版本要求的智能手机，并且能够连接互联网
•用户愿意授予APP必要的权限（通知、语音、相机、相册）
•第三方大模型API服务稳定可用，响应时间符合要求
•云服务器和数据库服务稳定可靠，全年可用率≥99.9%
•项目团队人员齐全，能够按时完成开发任务
•用户能够理解并接受AI生成日程可能存在的误差，愿意手动修正

3.2 客户端层架构
客户端采用hubuilderx框架实现模块化开发，依托框架的组件化、跨端适配特性，分为6个核心模块：
1.用户模块：负责用户注册、登录、游客模式、个人信息管理、数据同步设置
2.日历模块：负责日历视图展示（天/周/月）、日程的增删改查、分类管理、完成状态标记
3.提醒模块：负责日程提醒设置、晨间推送配置、提醒方式选择、通知触发
4.动态模块：负责动态的发布、查看、编辑、删除、时间线展示、搜索
5.AI模块：负责文字/语音输入、AI请求发送、生成结果预览、日程确认添加
6.基础模块：负责本地数据存储、网络请求封装、系统权限管理、异常处理
3.3 服务端层架构
服务端采用微服务架构，拆分为4个独立的服务：
1.用户服务：处理用户注册、登录、认证、个人信息管理、权限控制
2.日程服务：处理日程的增删改查、分类管理、提醒时间计算、数据同步
3.动态服务：处理动态的发布、存储、查询、编辑、删除、图片上传、日程关联
4.AI代理服务：负责与第三方大模型API的交互，封装请求参数，解析返回结果，进行错误处理
3.4 基础设施层
1.MySQL数据库：存储用户信息、日程数据、分类数据、动态元数据、日程关联关系
2.腾讯云COS：存储用户上传的动态图片和其他文件
3.消息推送服务：实现APP的通知推送功能，支持离线消息
4.第三方API：豆包大模型API（智能日程生成）、短信验证API（用户注册）
3.5 数据流向
1.用户操作：用户在客户端进行操作（如添加日程、发布动态）
2.本地存储：客户端首先将数据保存到本地数据库，确保离线可用
3.网络请求：客户端将数据通过HTTPS协议发送到服务端对应接口
4.服务处理：服务端对数据进行校验和处理，保存到数据库或云存储
5.数据同步：服务端将处理结果返回给客户端，客户端更新本地数据
6.AI交互：当用户使用AI功能时，客户端将输入发送到AI代理服务，AI代理服务调用第三方API，将结果返回给客户端

4.2 核心用例详细描述
4.2.1 AI生成日程用例
•前置条件：用户已登录，网络连接正常
•基本流程：
a.用户点击首页"AI生成"浮动按钮
b.进入AI交互界面，显示输入框和语音输入按钮
c.用户输入文字描述或点击语音按钮进行语音输入
d.系统将输入内容发送到后端AI代理服务
e.AI代理服务调用豆包大模型API，传递优化后的提示词
f.大模型解析自然语言，提取任务标题、开始时间、结束时间、优先级等信息
g.AI代理服务将解析结果返回给客户端
h.客户端展示日程预览界面，显示生成的所有日程信息
i.用户确认无误后点击"添加到日历"按钮
j.系统将日程保存到本地和云端数据库，更新日历视图
k.提示"日程添加成功"
•异常流程：
￮网络错误：提示"请检查网络连接"，保留用户输入内容
￮AI服务不可用：提示"AI服务暂时不可用，请手动添加日程"
￮输入不明确：提示"您的描述不够清晰，请补充任务时间和内容"
￮生成结果错误：允许用户在预览界面直接修改日程信息
4.2.2 晨间待办推送用例
•前置条件：用户已开启晨间推送功能，设置了推送时间
•基本流程：
a.系统在用户设置的推送时间前5分钟，查询用户当日所有未完成的日程
b.生成当日待办清单，按照优先级排序
c.调用系统通知服务，推送待办清单到用户手机通知栏
d.用户点击通知，直接进入APP的今日待办页面
•异常流程：
￮用户当日无待办：不推送通知
￮用户关闭了APP通知权限：提示"请开启通知权限以接收晨间推送"
4.2.3 日历月视图快速新增日程用例
•前置条件：用户已进入日历月视图页面
•基本流程：
a.用户点击任意日期格子
b.系统弹出新增日程表单，自动填充选中的日期
c.用户填写日程标题、时间、分类等信息
d.点击"保存"按钮
e.系统保存日程并更新日历视图，在对应日期格子内显示该日程的彩色小标识
•异常流程：
￮用户未填写标题：提示"请输入日程标题"
￮时间设置错误：提示"结束时间不能早于开始时间"
4.2.4 动态日程挂载用例
•前置条件：用户已进入发布动态页面
•基本流程：
a.用户输入动态文字内容，添加图片（可选）
b.点击"关联日程"按钮
c.系统弹出最近7天的日程列表供选择
d.用户选择要关联的日程
e.点击"发布"按钮
f.系统保存动态并建立与对应日程的关联关系
g.动态发布成功，在动态卡片上显示关联的日程标题
•异常流程：
￮用户未选择日程：直接发布无关联的动态
￮关联的日程已被删除：提示"该日程已不存在"

5. 非功能需求描述
5.1 性能要求
•APP冷启动时间≤2秒，热启动时间≤0.5秒
•页面切换时间≤0.5秒
•AI生成日程响应时间≤3秒（网络正常情况下）
•通知推送时间误差≤1分钟
•支持1000用户同时在线使用
•单条动态发布时间≤2秒（图片≤5张，单张≤5MB）
•数据同步时间≤5秒（100条日程以内）
•日历月视图加载时间≤1秒（30天数据）
5.2 安全性要求
•用户密码采用BCrypt算法加密存储，不存储明文密码
•所有数据传输采用HTTPS协议加密
•用户数据严格隔离，不同用户之间无法访问对方数据
•提供账号注销功能，注销后彻底删除用户所有数据
•每日自动备份数据库，备份数据保留7天
•防止SQL注入、XSS攻击等常见安全漏洞
•第三方API密钥加密存储，不在客户端硬编码
5.3 可靠性要求
•系统全年可用率≥99.9%
•服务器故障恢复时间≤30分钟
•数据丢失率≤0.01%
•支持离线操作，联网后自动同步数据
•异常操作有明确的错误提示，不会导致APP崩溃
5.4 可扩展性要求
•采用模块化设计，各模块之间低耦合高内聚
•服务端接口采用RESTful风格，便于后续扩展
•数据库设计预留扩展字段，支持新增功能
•支持横向扩展，通过增加服务器节点提升系统性能
•预留第三方服务集成接口，便于后续接入其他AI模型或服务
5.5 界面要求
•整体风格：简洁清新的扁平化设计，符合年轻用户审美
•主色调：浅蓝色（#4A90E2），辅助色：浅绿色（#50C878）、浅橙色（#FFA500）
•布局：底部导航栏设计，分为"日历"、"提醒"、"动态"、"我的"四个tab
•交互：操作简单直观，符合移动端使用习惯，提供明确的视觉反馈
•字体：采用系统默认字体，字号适中，保证可读性
•适配：支持不同分辨率的Android手机，适配全面屏和刘海屏
•日历格子：大小统一，小标识宽度不超过格子宽度的80%，最多显示3个标识，超出部分显示"..."；小标识高度为日历格子高度的1/4，颜色与对应日程分类颜色完全一致，相邻标识间距为2dp，标识圆角为2dp，确保视觉清晰不重叠
•提醒中心时间轴：时间轴主线为浅灰色（#E5E5E5），线宽2dp；今日剩余待办节点为浅蓝色（#4A90E2），直径8dp，节点旁标注具体时间（格式为HH:MM），文字为黑色（#333333）；未来待办节点为浅绿色（#50C878），直径6dp，标注日期（格式为MM-DD）；已过期非截止性任务节点为浅灰色（#999999），直径6dp，任务文字为浅灰色（#999999），整体排版左对齐，时间轴与文字间距为8dp
•动态日期筛选器：筛选器高度为44dp，宽度与屏幕同宽，横向滚动无滚动条；日期按钮为圆角矩形（圆角8dp），未选中状态背景为白色、文字为灰色（#666666），选中状态背景为浅蓝色（#4A90E2）、文字为白色；每个日期按钮宽度为40dp，间距为8dp，显示格式为"MM-DD"，今日日期按钮添加浅橙色（#FFA500）边框，边框宽度1dp，便于用户快速定位今日动态
•日历周视图：在顶部月份选择器旁提供视图切换（天/周/月），周视图以当前选中日期为基准展示该周7天，周格展示同月小标识与颜色，点击周内日期可快速新增日程，周格按照屏幕宽度等分7列展示
•提醒中心已完成项显示：提醒页今日分区改为同时展示所有今日任务（包括已完成），已完成任务请使用划线（line-through）与降低不透明度（opacity:0.6）表示，已完成项仍保留编辑与删除入口
•生活动态卡片规范（v2.0竖版左侧日期布局）：动态卡片全面仿微信朋友圈UI布局，具体规范如下：
  - 页面结构：页面背景浅灰色(#ededed)，顶部仅保留"生活动态"标题栏，无日期横向筛选器
  - 卡片整体布局：白色背景卡片(圆角4px)，横向flex布局，左侧52px日期栏 + 右侧flex:1内容区，卡片间距8px
  - 左侧日期栏(52px)：上方显示日期MM-DD(15px，粗体，#333)，下方显示中文星期(11px，#999)，居中对齐
  - 头部区域：方形头像(38x38px，圆角4px) + 微信蓝昵称(14px，粗体，#576b95) / 相对时间(11px，灰色#9a9a9a)，头像与内容区缩进46px
  - 发布时间：采用相对时间格式——"刚刚"(<1分钟)、"X分钟前"(<1小时)、"X小时前"(<24小时)、"昨天"、"MM-DD HH:mm"
  - 正文文本：15px字号，行高1.55，颜色#1f1f1f，支持长文本折叠（超过140字符/6行显示"全文"按钮，点击展开，再次点击"收起"）
  - 图片九宫格：最大宽度220px，1张图为4:3大图模式(占170px宽)；2张两列(190px)；3张三列(220px)；4张两行两列；5-9张三列排列；图片间距2px，圆角2px，支持点击预览放大
  - 关联日程链接：浅灰(#f5f7fa)背景卡片式，左侧📎图标+日程标题(12px，#576b95)，圆角4px，最大宽度230px
  - 底部操作栏：0.5px顶部分割线(#e5e5e5) + "赞"/"评论"纯文字按钮(13px，#999)，间距20px
  - 空状态：居中展示📝图标(透明度0.4)+"还没有动态，记录点什么吧"文案(#999)
  - 发布按钮：微信绿(#07c160)圆形FAB(50x50px)，内嵌📷相机图标，阴影0 2px 12px rgba(7,193,96,0.3)
5.6 易用性要求
•支持游客模式，无需注册即可使用基本功能
•核心功能操作步骤不超过3步
•提供新手引导，帮助用户快速上手
•界面文字简洁明了，避免专业术语
•支持撤销操作，防止误删除重要数据
6. 软件原型
6.1 主要页面结构
1.日历页（首页）
￮顶部：月份选择器、今日按钮、视图切换按钮（天/周/月）
￮中间：最近30天日历网格，每个日期格子内以横向彩色小标识展示当日任务（小标识颜色与日程分类颜色一致）
￮底部：今日待办列表，按优先级排序
￮右下角：浮动"添加"按钮和"AI生成"按钮
￮交互：点击任意日期格子直接进入新增日程页面，自动填充该日期
2.提醒页（提醒中心）
￮顶部：标题、设置按钮
￮中间：时间轴形式展示所有待办事项
▪第一部分：今日剩余待办，按时间先后顺序置顶展示，高优先级任务标红
▪第二部分：未来待办，按日期分组展示
▪第三部分：已过期非截止性任务，自动置灰并移至底部
￮底部：晨间推送设置区域，可设置推送时间和开关
3.动态页（仿微信朋友圈 v2.0竖版左侧日期布局）
￮顶部：仅标题"生活动态"（灰色背景，无日期筛选器）
￮中间：竖版排列动态卡片（白色背景，圆角4px，卡片间距8px），每条采用左右两栏布局：
  - 左侧日期栏(52px)：上方MM-DD日期(15px粗体)，下方中文星期(11px灰色)，居中对齐
  - 右侧内容区：方形头像(38px) + 微信蓝昵称(14px) + 相对时间(11px灰色)
  - 正文内容(15px，#1f1f1f)，超长文本折叠显示"全文"/"收起"按钮
  - 图片九宫格（1张4:3大图170px，2-9张正方形网格220px，2px间距）
  - 关联日程链接（浅灰卡片样式，如有）
  - 底部"赞"/"评论"纯文字操作栏(0.5px细线分隔)
￮右下角：微信绿(#07c160)圆形FAB按钮(50x50px)，内嵌📷相机图标
￮交互：点击图片可全屏预览并左右滑动；点击"全文"/"收起"展开折叠文本；点击关联日程跳转到对应日程详情
￮展示全部动态（按时间倒序），不再按日期筛选
4.我的页
￮顶部：个人信息区域（头像、昵称）
￮中间：功能列表（分类管理、通知设置、数据同步、关于我们）
￮底部：版本信息、账号注销按钮
6.2 核心交互流程
1.AI生成日程：点击首页AI按钮 → 输入需求 → 预览生成结果 → 确认添加到日历
2.发布动态并挂载日程：点击动态页发布按钮 → 输入文字/添加图片 → 点击"关联日程" → 选择对应日程 → 点击发布 → 动态出现在时间线并显示关联日程
3.设置提醒：编辑日程 → 点击"添加提醒" → 选择提醒时间和方式 → 保存 → 到时间自动推送通知
4.标记日程完成：在日历页或待办列表中点击日程右侧的复选框 → 日程变为灰色并划掉 → 自动从今日待办列表移除
5.日历快速新增日程：在月视图中点击目标日期格子 → 填写日程信息 → 点击保存 → 对应日期格子显示该日程的彩色小标识
